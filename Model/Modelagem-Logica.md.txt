1. Introdução
Este documento detalha a modelagem lógica do banco de dados para um sistema de loja de roupa, contemplando as tabelas essenciais para controle de produtos, clientes, vendas, estoque e fornecedores.

2. Tabelas e Atributos

2.1 Tabela: Cliente
Atributo	Tipo	Descrição	Restrição
id_cliente	INT	Identificador único	Chave Primária (PK)
nome	VARCHAR(100)	Nome completo do cliente	NOT NULL
email	VARCHAR(150)	Email do cliente	UNIQUE
telefone	VARCHAR(20)	Telefone para contato	
endereco	VARCHAR(255)	Endereço	

2.2 Tabela: Produto
Atributo	Tipo	Descrição	Restrição
id_produto	INT	Identificador único	Chave Primária (PK)
nome	VARCHAR(100)	Nome do produto	NOT NULL
descricao	TEXT	Descrição detalhada	
preco	DECIMAL(10,2)	Preço unitário	NOT NULL
id_fornecedor	INT	Referência ao fornecedor	Chave Estrangeira (FK)
categoria	VARCHAR(50)	Categoria do produto	

2.3 Tabela: Fornecedor
Atributo	Tipo	Descrição	Restrição
id_fornecedor	INT	Identificador único	Chave Primária (PK)
nome	VARCHAR(100)	Nome do fornecedor	NOT NULL
telefone	VARCHAR(20)	Telefone para contato	
email	VARCHAR(150)	Email do fornecedor	

2.4 Tabela: Venda
Atributo	Tipo	Descrição	Restrição
id_venda	INT	Identificador único	Chave Primária (PK)
id_cliente	INT	Referência ao cliente	Chave Estrangeira (FK)
data_venda	DATETIME	Data e hora da venda	NOT NULL
valor_total	DECIMAL(10,2)	Valor total da venda	NOT NULL

2.5 Tabela: ItemVenda
Atributo	Tipo	Descrição	Restrição
id_item_venda	INT	Identificador único	Chave Primária (PK)
id_venda	INT	Referência à venda	Chave Estrangeira (FK)
id_produto	INT	Referência ao produto	Chave Estrangeira (FK)
quantidade	INT	Quantidade vendida	NOT NULL
preco_unitario	DECIMAL(10,2)	Preço unitário na venda	NOT NULL

2.6 Tabela: Estoque
Atributo	Tipo	Descrição	Restrição
id_estoque	INT	Identificador único	Chave Primária (PK)
id_produto	INT	Referência ao produto	Chave Estrangeira (FK)
quantidade	INT	Quantidade em estoque	NOT NULL

3. Chaves e Relacionamentos
Cliente possui chave primária id_cliente.
Fornecedor possui chave primária id_fornecedor.
Produto possui chave primária id_produto e referencia o fornecedor via id_fornecedor.
Venda referencia o cliente (id_cliente) responsável pela compra.
ItemVenda representa os produtos vendidos em cada venda, relacionando Venda e Produto com as chaves estrangeiras id_venda e id_produto.
Estoque controla a quantidade disponível de cada produto
